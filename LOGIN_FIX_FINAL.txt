ISSUE
─────────────────────────────────────────────────────────────
After login, user sees "Login successful" toast but stays on
the login page instead of navigating to home.

ROOT CAUSE
─────────────────────────────────────────────────────────────
The App component was checking localStorage.getItem("token")
only once when it first rendered. When the user logged in and
the token was set, the App component didn't re-render to check
the new authentication state.

SOLUTION
─────────────────────────────────────────────────────────────
1. Added state management for authentication in App.js
2. Created custom 'authChange' event
3. Trigger event when token is set/removed
4. Listen for event and update state
5. Added PublicRoute component to redirect authenticated users

CHANGES MADE
─────────────────────────────────────────────────────────────

1. App.js:
   ✅ Added isAuthenticated state
   ✅ Added useEffect to listen for auth changes
   ✅ Created custom 'authChange' event listener
   ✅ Added PublicRoute component
   ✅ Wrapped login/register with PublicRoute

2. login.js:
   ✅ Dispatch 'authChange' event after setting token
   ✅ Works for both email/password and Google login

3. Navbar.js:
   ✅ Dispatch 'authChange' event on logout
